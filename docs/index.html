Comment on your page.
Claude reads it.

Claudback is a Chrome extension that lets you pin comments to elements on any web page — and a local MCP server that lets Claude read them and make the changes.

Get the extension claudback-mcp on npm

Iterate on your build without describing it

The main use case: you're building a site or prototype locally and iterating with Claude Code. Instead of screenshotting, or typing "the third button in the sidebar, no, the other sidebar" — you click the thing and say what you want. Each comment carries the exact element, so Claude knows precisely what you mean.

  1. Run your app locally and open it in Chrome. Click the Claudback icon and enable it for the tab.
  2. Pin comments to anything on the page: "make this bigger", "wrong green", "this overlaps on mobile".
  3. Ask Claude Code: check my claudback comments and make the changes. Claude reads each comment — with its element selector — through the MCP server.
  4. Comments resolve as Claude works through them. Reload, look at the result, drop the next round of comments.

It's not just for local builds — annotate any site to collect structured, element-anchored notes for Claude to work from: competitor teardowns, design reviews, content audits.

How it works

Claude MCP tools

The claudback-mcp server runs on your machine, started and stopped by Claude itself. It embeds a small HTTP collector that only listens on localhost. The extension syncs your comments to it; Claude reads them through MCP tools (get_comments, list_origins, resolve_comment, clear_comments).

Quick start

  1. Install the extension. Get Claudback from the Chrome Web Store (link coming soon). A setup guide opens automatically on install.
  2. Register the MCP server. For Claude Code: 
    claude mcp add claudback -- npx -y claudback-mcp
    Or for Claude Desktop, add to claude_desktop_config.json: 
    {
  "mcpServers": {
    "claudback": {
      "command": "npx",
      "args": ["-y", "claudback-mcp"]
    }
  }
}
  3. Pair. The server saves a pairing token to ~/.claudback/token on first run — paste it into the extension's setup page and you're connected.

Private by design

Local only

The collector binds to 127.0.0.1 — it is never reachable from the network. Your comments live in ~/.claudback/ on your own machine. No remote servers, no accounts, no analytics.

Paired & permissioned

Every request needs the pairing token, and only the extension's own origin passes CORS — a random web page can't write into your comment store. The extension is off by default and enabled per tab, with per-site permissions granted only when you ask.

Pull, not push

Comments never enter Claude's context automatically — Claude reads them only when you ask it to call the tools, and each comment is wrapped in an untrusted-data envelope. Full threat model in the repo.

Requires Node 20+ and Claude Code or Claude Desktop.
