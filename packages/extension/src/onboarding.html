Welcome to Claudback

Point at your page. Claude reads it.

Claudback lets you pin comments to elements on any web page — a button that's the wrong colour, a layout that breaks, copy that needs a rewrite. Claude reads your comments through a local MCP server and makes the changes.

The main use case: run your site or prototype locally, click the things you want changed and say why, then ask Claude Code to check your Claudback comments. No screenshots, no describing where things are — the comment carries the exact element.

Everything stays on your machine: comments go to a local collector on 127.0.0.1, never to a remote server.

Install the server

Claude starts and stops the server itself — you just register it once. Pick your client:

Run this in your terminal:

claude mcp add claudback -- npx -y claudback-mcp

Add this to your claude_desktop_config.json:

{
  "mcpServers": {
    "claudback": {
      "command": "npx",
      "args": ["-y", "claudback-mcp"]
    }
  }
}

On first run the server prints a pairing token and saves it to ~/.claudback/token — you'll paste that in the next step.

Pair the extension

Start a Claude session (or run npx claudback-mcp once in a terminal), then paste the token from ~/.claudback/token:

You're set — here's the loop

  1. Open the page you're working on and click the Claudback icon → Enable for this tab (grant the per-site permission when prompted).
  2. Use the floating button to pin comments to elements — click the thing, say what you want.
  3. Ask Claude: check my claudback comments and make the changes.
  4. Resolved comments clear automatically as Claude works through them.

You can reopen this guide any time from the extension's options page.
